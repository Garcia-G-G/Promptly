# Design Spec: Rails 8 App Scaffold + Auth + Workspace Model

- **Date**: 2026-04-14
- **Task**: 01-rails-app-scaffold
- **Status**: Approved

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Ruby version | 3.4 | Latest stable, greenfield project |
| Rails generation | `--skip-git` into `app/` | Avoids git conflicts in monorepo |
| Devise confirmable | Include, auto-confirm in dev | Matches spec, no dev friction |
| API key hashing | SHA-256 + per-workspace salt | Fast lookup, sufficient security for high-entropy keys |
| Build approach | Single sequential branch | Everything is interdependent in a scaffold |

## 1. Rails App Generation

```bash
rails new app --database=postgresql --skip-jbuilder --skip-git --name=Promptly
```

Rails 8 defaults retained: Propshaft, Importmap, Turbo, Stimulus, Solid Queue, Solid Cache, Solid Cable.

## 3. Data Models & Migrations

Migration order (sequential, each depends on previous):

### 3.1 Users (Devise)

Generated by Devise. Devise modules: `database_authenticatable`, `registerable`, `recoverable`, `rememberable`, `validatable`, `confirmable`.

Additional column: `name:string`.

Model: `normalizes :email, with: -> { _1.strip.downcase }`

### 3.2 Workspaces

```
name:string
slug:string (unique index)
plan:string (default: "starter")
stripe_customer_id:string
stripe_meter_id:string
owner_id:references(User)
timestamps
```

Model:
- `normalizes :slug, with: -> { _1.strip.downcase }`
- `validates :slug, presence: true, uniqueness: true, format: { with: /\A[a-z0-9-]+\z/ }`
- `belongs_to :owner, class_name: "User"`
- `has_many :memberships, dependent: :destroy`
- `has_many :users, through: :memberships`
- `has_many :projects, dependent: :destroy`
- `has_many :api_keys, dependent: :destroy`

### 3.3 Memberships

```
workspace_id:references
user_id:references
role:string (default: "developer")
timestamps
```

Unique composite index on `[workspace_id, user_id]`.

Model:
- `enum :role, { owner: "owner", admin: "admin", developer: "developer", viewer: "viewer" }`
- `validates :role, inclusion: { in: %w[owner admin developer viewer] }`
- `validates :user_id, uniqueness: { scope: :workspace_id }`

### 3.4 Projects

```
workspace_id:references
name:string
slug:string
timestamps
```

Unique composite index on `[workspace_id, slug]` (scoped uniqueness).

Model:
- `normalizes :slug, with: -> { _1.strip.downcase }`
- `validates :slug, presence: true, uniqueness: { scope: :workspace_id }, format: { with: /\A[a-z0-9-]+\z/ }`
- `belongs_to :workspace`

### 3.5 ApiKeys

```
workspace_id:references
name:string
key_prefix:string (first 8 chars, for UI display)
key_digest:string (SHA-256 hash, indexed)
last_used_at:datetime
revoked_at:datetime
timestamps
```

Index on `key_digest` for fast lookup.

Model:
- `belongs_to :workspace`
- Hashing: `Digest::SHA256.hexdigest("#{workspace.id}:#{raw_key}")` — workspace ID serves as the salt
- Lookup: query by `key_digest` (indexed), no need to iterate
- `key_prefix` stores first 8 chars (e.g., `pk_a3f2b1`) for identification in UI
- Full raw key shown only once at creation time, never stored

## 4. Controllers & Routes

### 4.1 Web Controllers

**`ApplicationController`:**
- `before_action :authenticate_user!`
- `helper_method :current_workspace`
- `current_workspace` resolves from `params[:workspace_slug]`, verifies membership
- Returns 404 (not 403) if no access — don't leak workspace existence

**`MarketingController#index`:**
- Public landing page, root route
- `skip_before_action :authenticate_user!`

**`WorkspacesController`** (`new`, `create`, `show`):
- On create, auto-creates `owner` Membership for `current_user`

**`ProjectsController`** (`index`, `show`, `new`, `create`):
- Nested under workspace, scoped through `current_workspace`

### 4.2 API Controllers

**`Api::V1::BaseController`:**
- Inherits from `ActionController::API` (not `ApplicationController`)
- `skip_forgery_protection`
- `before_action :authenticate_api_key!` — stub returning 501
- JSON-only responses

**`Api::V1::PromptsController`:**
- Placeholder `resolve` and `log` returning `{ status: "not_implemented" }` with 501

### 4.3 Routes

```ruby
root "marketing#index"
devise_for :users

resources :workspaces, param: :slug, only: [:new, :create, :show] do
  resources :projects, param: :slug, only: [:index, :show, :new, :create]
end

namespace :api do
  namespace :v1 do
    post "prompts/:slug/resolve", to: "prompts#resolve"
    post "prompts/:slug/log",     to: "prompts#log"
  end
end
```

## 5. Configuration

### 5.1 Initializers

- `rack_attack.rb` — throttle `/api/**` at 1000 req/min per IP
- `secure_headers.rb` — HSTS, CSP (allow Turbo/Stimulus), X-Frame-Options DENY
- `strong_migrations.rb` — target Postgres 16
- `devise.rb` — Devise installer default

### 5.2 Database

`config/database.yml` uses `ENV["DATABASE_URL"]` with dev defaults (`promptly_development`, `promptly_test`).

## 7. Seeds

```ruby
  password: ENV.fetch("SEED_OWNER_PASSWORD", "promptly-dev"),

owner.confirm

workspace = Workspace.create!(name: "Demo", slug: "demo", owner: owner)
Membership.create!(workspace: workspace, user: owner, role: :owner)

```
